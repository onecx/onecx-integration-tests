= OneCX Integration Test Runner

OneCX Integration Tests is a TypeScript-based CLI that uses the `testcontainers` npm library — a Node.js port of the open-source Testcontainers framework — to start and orchestrate Docker containers for a complete OneCX platform stack. It validates configuration before startup, manages the full container lifecycle, and optionally runs End-to-End (E2E) tests as the final step. The runner works in local development environments and in CI pipelines and produces reproducible, inspectable run artifacts on every execution.

== Purpose

Use this module to:

* Start core, service, BFF, and UI containers in a predictable order
* Validate configuration before startup
* Run health checks and optional heartbeat monitoring
* Execute an E2E container as the final runtime step when configured
* Export runtime metadata and structured artifacts for debugging and reporting

The version semantics for docs and runtime contracts are strict:

* `main` means the latest build from the main branch
* Ambiguous or missing tag resolution must fail fast

== Audience

This documentation targets:

* Integrators who configure platform startup through `platform.json`
* Test engineers who run E2E and inspect artifacts
* Platform maintainers who extend orchestration behavior
* CI maintainers who integrate workflow execution
